Scripting / Instructions

IExecutionSuscriber

Lightweight notification interface for enemies and interactables that need to react to each instruction step. Implement it on any component in the enemy's hierarchy — the EnemyController discovers and registers all subscribers automatically at initialization.

Interface

MethodDescription
Execute(ExecutionContext context)Called every time the owning enemy executes an instruction step. Use it to trigger animations, sounds, projectiles, or any side-effect that should accompany the enemy's action.

How registration works

You do not need to register subscribers manually. When a level starts, EnemyController.Initialize() calls GetComponentsInChildren<IExecutionSuscriber>() on the enemy's GameObject and wires every result into the internal OnExecute event:

// inside EnemyController.Initialize() — runs automatically
IExecutionSuscriber[] suscribers = GetComponentsInChildren<IExecutionSuscriber>();
foreach (IExecutionSuscriber s in suscribers)
    OnExecute += s.Execute;

Each time the enemy's current instruction executes, EnemyController invokes OnExecute, which calls Execute() on every registered subscriber in order.

To add a subscriber to an enemy: attach a MonoBehaviour that implements IExecutionSuscriber to the enemy's root GameObject or any of its children. No other wiring is required.

IExecutionSuscriber vs ITaskReceiver

IExecutionSuscriberITaskReceiver
OwnershipReacts to the enemy's own actionSchedules an independent parallel task
TimingSynchronous — fires inline during instruction executionAsynchronous — runs concurrently alongside player instructions
Use whenYou need a fire-and-forget side-effect (sound, VFX, projectile) tied to the enemy's actionThe object needs its own coroutine that runs in parallel every turn
RegistrationAuto-discovered via GetComponentsInChildrenMust be registered with the runner explicitly

Example

A subscriber that plays a sound clip each time the enemy executes an action:

using UnityEngine;

namespace LoopAdventure
{
    public class ActionSoundSubscriber : MonoBehaviour, IExecutionSuscriber
    {
        [SerializeField] AudioClip actionClip;
        AudioSource audioSource;

        void Start()
        {
            audioSource = GetComponentInParent<AudioSource>();
        }

        public void Execute(ExecutionContext context)
        {
            if (actionClip != null)
                audioSource.PlayOneShot(actionClip);
        }
    }
}

Attach this component to the enemy's root GameObject or any child. The EnemyController will pick it up automatically — no further wiring needed.
