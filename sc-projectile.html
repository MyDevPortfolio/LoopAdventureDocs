Scripting / Components

Projectile

A self-propelled prefab that travels in a straight line and applies damage on impact. Spawn and fire it with Shoot() from inside a ranged instruction's Tick() coroutine.

Inspector Fields

FieldTypeDescription
speedfloatTravel speed in world units per second. Default: 4.
damageintHit points removed from the first Health component hit. Default: 1.
shootSoundAudioClipPlayed once when the projectile is fired.
hitSoundAudioClipPlayed when the projectile hits a target.

Methods

MethodDescription
Initialize(ExecutionContext context, ExecutionContextElement parent)Calls base.Initialize() and registers this projectile with context.AddRuntimeElement(this) so it is cleaned up when the level resets. Called automatically by Shoot — do not call directly.
Shoot(ExecutionContext context, Direction direction, Vector3 position, Quaternion rotation, Transform parent = null) → ProjectileInstantiates a copy of the projectile prefab at position with rotation, mirrors the sprite scale to face the direction, and begins movement. Returns the new instance. Call this from your instruction's Tick() coroutine.

Usage example

protected override IEnumerator Tick(ExecutionContext ctx)
{
    Projectile p = projectilePrefab.Shoot(
        ctx,
        ctx.player.direction,
        ctx.player.transform.position,
        Quaternion.identity
    );
    yield return new WaitUntil(() => p == null);
}

The returned instance becomes null when it hits something or leaves the map, so yielding until it is destroyed gives the instruction a clean end condition.

ℹ️
The built-in ShootSuscriber IExecutionSuscriber wraps Shoot for use on enemies. If you only need enemy ranged attacks, you do not need to call Shoot directly — just attach ShootSuscriber to the enemy and assign the projectile prefab.
